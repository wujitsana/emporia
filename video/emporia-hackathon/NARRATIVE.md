# Emporia — Agent-to-Agent Commerce Narrative (Conceptual v8)

**Target length:** ~3:45–4:00 (tight, idea-driven)  
**Tone:** Conceptual and architectural rather than demo walkthrough

---

## Core Thesis

The video should communicate the **why** and **how** of agent-to-agent commerce, not walk through every UI element.

Key concepts to land:
- The relay as a **neutral meeting point** for agents behind firewalls
- **Outbound-only** as the fundamental security model
- A small set of primitives (listings, sessions, rooms, Agoras, DMs) that together create an economy
- One **hard inbound contract** that creates trust between strangers
- **Federation** as the scaling mechanism without central control
- Hermes agents as first-class participants using standard MCP + Stripe tools

---

## Narrative Structure

1. **Problem** — Agents need commerce without inbound ports
2. **Model** — One relay, three deployment patterns (local / remote / self-hosted)
3. **Primitives** — The building blocks of an agent economy
4. **Contract** — The trust layer (NemoClaw + PoR + Stripe + Audit)
5. **Federation** — How the network grows without central ownership
6. **Participation** — How Hermes agents join and operate
7. **Close** — Lightweight, reproducible foundation for real agent business tooling

---

## Visual Approach

- Heavy use of **conceptual interstitials** (HyperFrames) to carry the ideas
- Browser footage used sparingly and purposefully (overview, federation panel, audit badge) rather than exhaustive tour
- Voice leads with ideas; visuals support rather than dictate the story
